# AGENTS.md — заметки для агента, дорабатывающего этот репозиторий

Ориентир для будущих сессий. Не дублирует README — фиксирует неочевидное:
устройство, конвенции, гочи, как проверять. Держись этого при доработке.

## Что это

`just-searxng-skill` — распространяемый **скилл для ИИ-агентов**: локальный
веб-поиск (SearXNG) + чтение веб-страниц (curl + Mozilla Readability). Ставится
пользователям через `npx skills add AleshaIdetRvat/just-searxng-skill`. Этот
git-репозиторий — источник правды; «релиз» = `git push` (владелец — автор скилла,
не потребитель).

## Критическая раскладка (не сломай)

Сам скилл живёт в подпапке **`searxng/`**, а не в корне репозитория. Причина:
`npx skills` считает корневой `SKILL.md` «однофайловым скиллом» и копирует
**только** его, теряя `scripts/` и `config/`. Скилл в именованной подпапке
копируется целиком. **Не переноси `SKILL.md` в корень.**

## Карта репозитория

- `README.md` — **русский**, для владельца: установка, конфиг, внутренности, диагностика.
- `AGENTS.md` — этот файл.
- `searxng/SKILL.md` — **английский**, для агента-потребителя. Тонкий: «инфра готова, вот как пользоваться». Фронтматтер `description` — триггер (см. ниже).
- `searxng/ABOUT.md` — английский, для человека со вопросом «что это».
- `searxng/config/settings.yml` — шаблон настроек SearXNG (`__SECRET_KEY__` подставляется при первом запуске).
- `searxng/scripts/`
  - `search.sh` — вход поиска. Гарантирует живой SearXNG через `ensure-searxng.sh`, шлёт `GET /search?format=json`, форматирует (jq → python3 fallback).
  - `ensure-searxng.sh` — идемпотентный авто-подъём локального контейнера SearXNG. **stdout = base URL, stderr = ошибки/подсказки.** Коды: `0` ок / `3` нет рантайма / `4` не поднялся / `5` нет curl.
  - `fetch.sh` — чтение страницы: curl → Node-экстрактор. Сам ставит зависимости при первом вызове. Коды: `2` аргументы / `4` сеть-HTTP-не-HTML / `5` нет Node/deps.
  - `reader/extract.mjs` — Node: HTML со stdin → чистый текст / `--html` / `--json` (Mozilla Readability + linkedom).
  - `reader/package.json` — зависимости экстрактора. `node_modules/` в `.gitignore`.

## Конвенции

- **Язык.** `SKILL.md`, `ABOUT.md`, скрипты и все их сообщения — по-английски. `README.md` и этот файл — по-русски (для владельца).
- **Коммиты.** Conventional Commits на русском. В репо есть скилл `/git-commit` — используй его. История без Claude-трейлеров — не добавляй их.
- **Bash-скрипты.** `set -euo pipefail`; `SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"`; **stdout = данные, stderr = сообщения**; действенные ошибки с кодами возврата; `curl --data-urlencode` для параметров запроса (никогда не интерполируй ввод прямо в URL).

## Философия (держи при доработке)

- **Тонкий SKILL.md, прогрессивное раскрытие.** Не клади в SKILL.md failure-mode инструкции — скрипты сами печатают действенные сообщения на stderr, а SKILL.md лишь говорит «прочитай, что печатает скрипт, и сделай». Детали установки/диагностики живут в README, не в SKILL.md.
- **Авто-setup при первом использовании.** Контейнер SearXNG и Node-зависимости reader поднимаются сами на первом вызове; человек не вовлечён (исключение: совсем нет рантайма/Node → скрипт печатает разовую команду и выходит, ничего не ставя молча).
- **`description` во фронтматтере SKILL.md — это триггер.** Агент решает, брать ли скилл, по нему (до загрузки тела). Всё про «когда срабатывать» и «предпочитать этому, а не curl» правится там.
- **Минимализм («just»).** Без тяжёлых зависимостей. Чтение страниц — обычный curl + Readability, а не headless-браузер.

## Как проверять изменения (реально запускать, не только читать)

- Синтаксис: `bash -n searxng/scripts/*.sh` и `node --check searxng/scripts/reader/extract.mjs`.
- Поиск end-to-end: `searxng/scripts/search.sh "test" -n 3` (первый раз поднимет контейнер, ~30–60с).
- Чтение end-to-end: `searxng/scripts/fetch.sh "https://en.wikipedia.org/wiki/Readability" -n 800` (первый раз поставит npm-зависимости).
- Экстрактор изолированно, без сети: `node searxng/scripts/reader/extract.mjs --mode json < file.html`.

## Гочи

- **Node ставится через nvm с ленивой загрузкой:** `node` — это shell-функция, её нет в PATH неинтерактивного скрипта. `fetch.sh` это обходит (находит nvm-дефолт или новейшую версию и добавляет её bin в PATH). Трогаешь запуск Node — не сломай этот фолбэк.
- **`node_modules` не коммитим** (в `.gitignore`): при обновлении скилла к пользователю приезжает только `package.json`, а deps ставятся при первом `fetch.sh`. Это норма, не баг.
- **SearXNG-конфиг — три обязательные вещи**, иначе поиск «молча не работает»: `search.formats` со значением `json`; `server.limiter: false`; устойчивый набор движков поверх дефолтов (**bing, mojeek, presearch** — почти не троттлятся). Порт хоста по умолчанию **8888**, внутри контейнера 8080. `use_default_settings: true` → мы переопределяем только эти пункты, список `engines` сливается с дефолтным по имени.
- **Env-переменные:** `SEARXNG_URL` (использовать готовый инстанс как есть), `SEARXNG_PORT`, `SEARXNG_IMAGE`, `SEARXNG_CONTAINER`, `SEARXNG_SKILL_HOME`, `SEARXNG_VERBOSE`.

## Референс

Идея чтения страниц навеяна `browser-search` (github.com/Johell1NS/browser-search) —
трёхуровневый SearXNG + Camofox + CloakBrowser. Мы взяли только Readability, без
браузеров: `curl` + Mozilla Readability на `linkedom`-DOM, одна лёгкая Node-зависимость.
